Linked List

Key point:

  • When you want to dereference a ListNode, make sure it is not a null pointer
  • NEVER EVER lose the control of the head pointer of the LinkedList
  • Only use dummy head when we want to build a LinkedList from scratch, to avoid null pointer dereference
  • Maintain a tail pointer when we want to frequently append a new Node to the end of LinkedList

Array VS Linked List:

  • Memory Layout
    • Array: consecutive allocated memory space, no overhead
    • Linked List: con- consecutive, overhead of of multiple objects with the ‘next’ reference
  • (Random) access time:
    • Array: O(1)
    • LinkedList: O(n)
  • Search time:
    • Non sorted: Array: O(n); LinkedList: O(n)
    • Sorted: Array: O(logn); LinkedList O(n)

Iterative Way:

  • assume the list start from n2 is already reversed: here we have … -> n3 -> n2;
  • we need to revert n1, we want n2 -> n1 -> null;
  • we can easily assess n2 by n1.next; since n1 is still point to n2,
